Introduction To Information System

What is Information System

Information System is a set of interrelated components working together to collect, retrieve, process, store and disseminate information for the purpose of facilitating planning, controlling, coordinating and decisions making.

Structures Of Information System

  • Planned or Structured Information Systems
  • Unplanned or Unstructured Information Systems
  • Manual or Automated Information Systems

Manual Information Systems

Are information systems which deal mostly with paper based documents and information within this system is undelivered and dependant on word of mouth communication channels.

Automated Information Systems

Are mostly planned systems which makes use of computer hardware and software amd other technologies to transform data resources into many varied information products for the use of consumers.

Advantages of Automated Information Systems

  • Automated Information Systems promotes efficiency and higher productivity.
  • Duplication is reduced to the minimum.
  • The speed for processing, retrieval and transmission of data is faster.
  • They grant multiple access and use of information at a time.

Disadvantages Of Automated Information Systems

  • With Automated Information Systems a high level of security is required.
  • It is cost effective
  • It require some level of expertise from both the custodian and the user.

Components of Information System

  1. Hardware
  2. Software
  3. Data
  4. People
  5. Procedures

Types Of Information System

  • Transaction Processing System
  • Office Automation System
  • Knowledge Work System
  • Decision Support System
  • Management Information System
  • Executive Support System

Introduction To Decision Support System

What is Decision Support System

It is a computer application design to analyzes data and provide the necessary information to managers for proper decision making in an organization.

Components Of Decision Support System

  • User Interface
  • Corporate database
  • Model Databases
  • Planning Language

Explains:

User interface: Is the component that allows you to combine your know-how with the storage and processing capabilities of the the computer.

Corporate Database: Is a data management component which performs the function of storing and maintaining the information that you want your decision support system to use.

Model Database: Is a type of data model that determines the logical structure of a database.

Planning Language: Consists of rules that can contrain possible solution as well as alternative solutions and methods for evaluating them.

Types Of Planning Language

  • General-purpose: It allows users to perform many routine tasks eg retrieving various data from a data-base or performing statistical analysis.
  • Special purpose: Are more limited in what they can do, but they usually do certain jobs such as forecasting, risk analysis and more better than the general purpose planning language.
